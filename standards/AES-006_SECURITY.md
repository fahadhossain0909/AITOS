# AES-006 — Security Standard

Version: 1.0

Status: Active

---

# Purpose

Security is an architectural responsibility.

Every engineering decision should consider security implications.

---

# Core Principles

- Least Privilege
- Defense in Depth
- Secure by Default
- Explicit Permissions
- Zero Trust Mindset

---

# Secret Management

Secrets must never be committed.

Examples:

- API Keys
- Passwords
- Tokens
- Certificates
- Private Keys

Secrets should use dedicated secret-management solutions.

---

# Dependency Management

Dependencies should be:

- Version controlled
- Regularly updated
- Vulnerability scanned
- Minimised

---

# Access Control

Access should follow:

- Role-based permissions
- Principle of least privilege
- Audit logging

---

# Data Protection

Sensitive information should be:

- Encrypted
- Authenticated
- Validated
- Logged appropriately

---

# Incident Response

Security incidents require:

- Identification
- Containment
- Investigation
- Recovery
- Documentation

---

# Security Reviews

Architecture reviews should include security assessment.

Critical components require periodic review.

---

# Principle

Security is continuous.

It is never "finished."
