# AES-001 — Documentation Standard

## Objective

Documentation is treated as a first-class engineering artifact.

---

## Rules

- Documentation precedes implementation.
- Every major feature requires documentation.
- Architecture changes require documentation updates.
- Public APIs require documentation.
- Research findings must be documented.

---

## Required Sections

Every engineering document should define:

- Purpose
- Scope
- Responsibilities
- Dependencies
- Limitations
- References

---

## Writing Principles

- Clear
- Concise
- Versioned
- Traceable
- Reviewable

---

## Quality Criteria

Documentation must be:

- Accurate
- Current
- Reproducible
- Consistent
