# Security Policy

## Supported Versions

During early development, only the latest development branch is supported.

| Version | Supported |
|----------|-----------|
| main | ✅ |
| develop | ✅ |
| older releases | ❌ |

---

# Reporting a Vulnerability

Please do not disclose security vulnerabilities publicly before they have been reviewed.

Instead:

1. Open a private security report (when available).
2. Contact the maintainers.
3. Provide reproduction steps.
4. Include affected components.
5. Suggest possible mitigations if known.

---

# Scope

Security reports may include:

- Authentication issues
- Authorization flaws
- Data exposure
- Dependency vulnerabilities
- Supply chain risks
- Secrets accidentally committed
- Unsafe execution behavior

---

# Response Process

Every report will be:

1. Acknowledged
2. Reviewed
3. Validated
4. Prioritized
5. Fixed
6. Documented

---

# Responsible Disclosure

Please allow reasonable time for investigation and remediation before making any public disclosure.

---

# Security Principles

AITOS follows these principles:

- Least privilege
- Defense in depth
- Secure by default
- Explicit permissions
- Continuous review
- Dependency management
- Secrets must never be committed to the repository.
