# Project Memory

Version: 1.0

---

The Memory Layer stores accumulated project knowledge that should persist across engineering iterations.

Unlike Context, Memory records project history.

Memory grows continuously.

## Institutional Memory Principles

Memory exists to prevent the project from repeatedly solving the same problems.

Project memory should preserve:

- Decisions
- Failures
- Research
- Design rationale
- Technical debt
- Evolution history
- Lessons learned

Context explains the present.

Memory explains the past.

Together they enable better future decisions.

It should never be rewritten unnecessarily.
