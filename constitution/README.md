# Constitution

## Purpose

The Constitution defines the non-negotiable principles that govern the AITOS project.

Unlike technical documentation, constitutional documents are expected to remain stable over long periods and should only change through a formal review process.

---

# Structure

This directory contains:

- PROJECT_CONSTITUTION.md
- ENGINEERING_CONSTITUTION.md
- AI_CONSTITUTION.md
- RESEARCH_CONSTITUTION.md
- GOVERNANCE.md

---

# Scope

The Constitution defines:

- Project values
- Engineering rules
- AI usage principles
- Research ethics
- Governance process

It does **not** define implementation details.

Those belong in:

- Architecture
- Requirements
- Standards
- Specifications

---

# Amendment Policy

Constitutional documents should only be modified when:

- Project goals fundamentally change.
- Governance requires revision.
- Engineering principles become obsolete.

Every amendment should be documented through an ADR or RFC before implementation.

---

# Guiding Principle

> Stable principles enable sustainable engineering.
