# Contributing to AITOS

Thank you for your interest in contributing to AITOS.

Our goal is to build a long-term, evidence-driven Financial Intelligence Platform through disciplined engineering and collaborative research.

---

# Core Principles

Every contribution should follow these principles:

- Evidence before opinion.
- Documentation before implementation.
- Simplicity before complexity.
- Reproducibility before optimization.
- Long-term maintainability over short-term speed.

---

# Development Workflow

Every significant change should follow this sequence:

1. Research
2. Specification
3. Architecture Review
4. Implementation
5. Testing
6. Documentation Update
7. Pull Request
8. Review
9. Merge

---

# Branch Strategy

- `main` — Stable production branch
- `develop` — Active development
- `feature/<name>` — New features
- `fix/<name>` — Bug fixes
- `docs/<name>` — Documentation updates
- `research/<topic>` — Research work

---

# Commit Convention

We follow Conventional Commits.

Examples:

- feat:
- fix:
- docs:
- refactor:
- test:
- chore:

Example:

docs(foundation): add project charter

---

# Pull Requests

Every Pull Request should include:

- Purpose
- Summary of changes
- Related issue (if applicable)
- Documentation updates
- Testing status
- Checklist completion

---

# Documentation Requirements

Documentation must be updated whenever:

- Architecture changes
- Requirements change
- Public APIs change
- Research conclusions change
- Standards change

---

# Code Review Principles

Reviews focus on:

- Correctness
- Readability
- Maintainability
- Security
- Performance
- Documentation quality

---

# Engineering Standards

All contributors should follow the standards defined in the `/standards` directory.

---

Thank you for helping build AITOS.
